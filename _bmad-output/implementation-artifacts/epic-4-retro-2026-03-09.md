## Challenges & Growth Areas

### Database Migration Gap (From Epic 2/3)

**Issue:** Vercel deployment failed with "relation 'hero' does not exist"

**Root Cause:** Collections and globals added in Epics 2 and 3 never had migrations generated. Payload handles schema automatically in local development, masking the issue until production deployment.

**Resolution:** Generated comprehensive migration in Story 4.2 covering all Epic 2-4 tables.

**Process Learning:** Generate migrations before deployment, not after. Local development hides migration issues.

### Lighthouse CI Workflow Broken

**Issue:** The accessibility CI workflow added in Epic 2/3 retrospective has been failing since deployment.

**Root Cause:** Workflow uses `setup-node` which looks for `package-lock.json`, `npm-shrinkwrap.json`, or `yarn.lock`. Project uses pnpm with `pnpm-lock.yaml`.

**Error:** `Dependencies lock file is not found`

**Resolution Needed:** Add `pnpm/action-setup` before `setup-node` in the workflow.

**Process Learning:** "Done" must mean verified working, not just file committed.

### Story Scope Overlap

- Stories 4.3 and 4.4 were mostly verification of work already completed in Story 4.2
- Suggests epic may have been over-decomposed
- Consider combining related acceptance criteria in future planning
